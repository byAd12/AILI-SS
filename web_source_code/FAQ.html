Preguntas y Respuestas

¿Quieres una prueba gratis del software?

  1. Contacta via e-mail con un miembro del equipo en la página de contacto y se te concederá una licencia de 3 días de expiración.

Guía para obtener AILI-SS

  1. Obtenga una clave contactando con el responsable mediante un e-mail.
  2. Descargar la aplicación en la página de Descargas. Es recomendable instalar la última versión estable.
  3. Inicie la aplicación ejecutando el archivo "AILI.exe" (Windows).
  4. Inicie la aplicación como administrador para poder crear el archivo de registros (.txt) correspondiente.
  5. Registre su clave después de aceptar los términos de uso.
  6. Configure la aplicación con el botón "Config.json".
  7. Disfrute de una variedad de herramientas útiles.

¿Cómo ejecutar AILI-SS en Linux?

  1. Descargue el fichero con formato zip.
  2. Click derecho en el archivo ⇨ Extraer.
  3. Haga click derecho en la carpeta donde se encuentra la aplicación ⇨ Abrir Terminal.
  4. Escriba en la terminal lo siguiente para darle permisos de ejecución: sudo chmod 750 ./AILI-SS
  5. Si es la primera vez usando AILI-SS ejecute en la terminal: sudo ./AILI-SS
  6. Si ya registró su clave, haga click derecho en el archivo con nombre "AILI-SS" ⇨ Ejecutar.

¿Qué hace AILI-SS al ejecutarse?

  1. Primero extrae del archivo comprimido (.exe o .bin) los archivos necesarios para ser ejecutado en una ruta temporal que el sistema operativo elija.
  2. Luego el archivo principal arranca con ayuda de los archivos necesarios para su correcto funcionamiento.
  3. Al verificar una clave se hace una consulta a una base de datos externa para verificar si la clave introducida coincide, después actualizará el archivo de registros (.txt).
  4. Al configurar Config.json se modifica un archivo temporal con el nombre mencionado que tiene como formato: {"Interfaz": null, "Direc_Red": null}
  5. Al borrar la clave con la que iniciaste sesión modifica elimina la configuración dada.
  6. Cuando cierres la aplicación también se eliminarán todas las configuraciones al hacerse en caché.

¿Es segura la aplicación?

  1. Sí, y le animamos a monitorizar la aplicación en máquinas virtuales como any.run.

¿Porqué el antivirus actúa?

  1. El archivo ejecutable contiene archivos python (.py) que por defecto, como en otros lenguajes, no se confía en ellos.
  2. El archivo ejecutable no está firmado con un certificado digital, por lo tanto no tiene autor y es una causa común de que no se confíe.
  3. Los navegadores advierten que no se ha descargado el archivo muchas veces como para que sea seguro.

¿Porqué AILI-SS requiere algunas dependencias para algunas funciones?

  1. Requerimos nmap únicamente para la función de escaneo de puertos de algún dispositivo de la red ya que integra ciertas funciones que ayudan a la realización de estas búsquedas.
  2. Requerimos el interpretador de python para las funciones de monitoreo de paquetes y detección de arp spoofing al tratarse de terminales que ejecutan una función de python (versión 3). Además se requieren unas librerías específicas para facilitar la ejecucción de estas funciones.
